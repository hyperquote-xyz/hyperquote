# Maker SDK

The Maker SDK (`packages/sdk-maker/`) provides everything needed to run an automated market maker bot for the HyperQuote Options RFQ protocol.

## Quick Start

```bash
cd packages/sdk-maker
npm install
npm run dev
```

This starts a maker bot that:
1. Connects to the relay WebSocket
2. Listens for RFQ broadcasts
3. Filters RFQs against configured parameters
4. Prices options using a stub Black-Scholes engine
5. Signs EIP-712 quotes and submits them to the relay

## Package Structure

```
packages/sdk-maker/
  src/
    types.ts       — Core types (Quote, RFQ, MakerConfig, etc.)
    eip712.ts      — EIP-712 domain + typed data (matches Solidity exactly)
    signQuote.ts   — Sign quotes with ethers.js v6
    verifyQuote.ts — Verify quote signatures
    nonce.ts       — On-chain nonce management
    pricing.ts     — Pricing engine interface + BSM stub
    risk.ts        — Risk checks (notional, tenor, delta, strike band)
    rfqHash.ts     — Deterministic RFQ ID computation
    main.ts        — Maker bot main loop
    index.ts       — Public API re-exports
  test/
    eip712.test.ts — EIP-712 cross-verification with Solidity
```

## Configuration

All config is loaded from environment variables in `main.ts`:

| Variable | Default | Description |
|----------|---------|-------------|
| `MAKER_PRIVATE_KEY` | Anvil key 0 | Maker wallet private key |
| `CHAIN_ID` | `31337` | Chain ID for EIP-712 signing |
| `ENGINE_ADDRESS` | Anvil default | OptionsEngine contract address |
| `RPC_URL` | `http://127.0.0.1:8545` | JSON-RPC endpoint |
| `RELAY_WS_URL` | `ws://127.0.0.1:8080` | Relay WebSocket URL |
| `WHYPE_ADDRESS` | `0x...01` | WHYPE token address |
| `USDC_ADDRESS` | `0x...02` | USDC token address |
| `USDH_ADDRESS` | `0x...03` | USDH token address |
| `USDT0_ADDRESS` | `0x...04` | USDT0 token address |
| `HYPE_SPOT_USD` | `25` | Current HYPE spot price (USD) |
| `HYPE_IV_BPS` | `8000` | Implied volatility (bps, 8000 = 80%) |
| `RISK_FREE_RATE_BPS` | `500` | Risk-free rate (bps, 500 = 5%) |

## EIP-712 Signing

The SDK matches the Solidity `QuoteLib.QUOTE_TYPEHASH` exactly:

```
Quote(address maker,address taker,address underlying,address collateral,
      bool isCall,bool isMakerSeller,uint256 strike,uint256 quantity,
      uint256 premium,uint256 expiry,uint256 deadline,uint256 nonce)
```

Domain:
```
name:              "HyperQuote Options"
version:           "1"
chainId:           <from config>
verifyingContract: <OptionsEngine address>
```

Cross-verification: The SDK test (`test/eip712.test.ts`) compares struct hashes, domain separators, and full digests against known Solidity test vectors generated by `forge test --match-contract EIP712CrossCheck -vv`.

## V1 Role Convention

In V1, `isMakerSeller` is always `false`:
- **Maker** = buyer (signs quotes, receives the NFT position)
- **Taker** = seller (executes quotes, locks collateral)

The maker bot signs quotes as the buyer. The user/taker who submitted the RFQ calls `OptionsEngine.execute()` on-chain.

## Risk Checks

The `risk.ts` module enforces:

| Check | Config Key | Default |
|-------|-----------|---------|
| Max notional per collateral | `maxNotionalPerCollateral` | $1M per token |
| Max tenor | `maxTenorSecs` | 90 days |
| Max strike deviation from spot | `maxStrikeDeviationPct` | 50% |
| Max delta per expiry bucket | `maxDeltaPerExpiry` | 100 |
| Min premium | `minPremium` | 0.001 per collateral |

## Pricing Engine

The default `StubPricingEngine` uses Black-Scholes with a simple vol surface:
- ATM vol: 80% (configurable)
- Skew: 0.5% vol per 1% OTM
- Spread: 2% of notional over fair value

For production, implement the `PricingEngine` interface with your own model.

## Nonce Management

The `NonceManager` class:
1. Reads the current on-chain nonce via `engine.nonces(maker)`
2. Provides monotonically increasing local nonces
3. Supports resync after on-chain `incrementNonce()` calls

Cancellation strategies:
- **Single quote**: Call `engine.cancelQuote(quote)` on-chain
- **Bulk cancel**: Call `engine.incrementNonce()` to invalidate all quotes with nonce < newNonce
- **Offchain**: Relay can mark quotes as withdrawn (not yet implemented in V1)
